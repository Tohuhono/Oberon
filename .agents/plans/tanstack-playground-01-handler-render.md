# TanStack Playground Slice 1: Handler + Public Render

## Goal

Wire the TanStack Playground to the Oberon public rendering and Handler plumbing without introducing
the CMS UI yet.

## Scope

In scope:

- `apps/newground` only
- public Page rendering through `@oberoncms/core/render`
- `/cms/api/*` server route forwarding to the composed Oberon Handler
- `/cms` returning 404 until the Provider/Login slice
- narrow smoke coverage for the TanStack route skeleton

Out of scope:

- `TanstackOberonProvider`
- login/authenticated browser state
- CMS UI pages
- TanStack Recipe or create-oberon-app propagation
- caching/revalidation design

## What to Build

- Replace the hardcoded `/` component with Oberon `Render` backed by `adapter` and `clientConfig`,
  following the Next.js Playground public render reference.
- Add a public catch-all TanStack route for Page paths if the file-route shape is clean. TanStack
  wildcard routes use `$.tsx` and expose `_splat` route params.
- Add a TanStack Start server route for `/cms/api/$` that forwards `GET`, `POST`, `PUT`, `PATCH`,
  and `DELETE` to the composed Oberon `handler`.
- Convert `/cms` from placeholder content to an intentional 404.
- Add route-level `head` metadata with `getMetaData` only if it is straightforward with TanStack's
  route `head` hook.

## Candidate Files

- `apps/newground/src/routes/index.tsx`
- `apps/newground/src/routes/$.tsx`
- `apps/newground/src/routes/cms.tsx`
- `apps/newground/src/routes/cms/api/$.ts`
- `apps/newground/src/routeTree.gen.ts` (generated by TanStack Router tooling)
- `apps/newground/playwright.config.ts`
- `dev/playwright/test/smoke/smoke.spec.ts` or a new smoke spec if app-specific expectations are
  needed

## Testing Plan

- Keep validation inside the approved root script: `pnpm test:e2e:new`.
- Preserve the homepage smoke test and make it pass because `/` renders the bootstrapped welcome
  Page.
- Adjust the TanStack smoke expectation so `/cms` returning 404 is accepted for this slice without
  changing Next.js Playground expectations.
- Add a narrow Handler smoke assertion that proves a `/cms/api/*` request reaches the Oberon
  Handler. The assertion should prove routing reaches the Handler, not that login/auth behavior is
  complete.

## Acceptance Criteria

- `/` returns 200 and renders the bootstrapped welcome Page through Oberon `Render`.
- An unknown public route still renders the TanStack not-found UI.
- `/cms` returns 404.
- `/cms/api/*` reaches the composed Oberon Handler and returns the expected Handler-level status for
  a chosen unsupported or missing action.
- `pnpm test:e2e:new` passes from the repo root.

## Risks and Mitigations

- Risk: the shared smoke spec currently assumes `/cms` is 200. Mitigation: split or parameterize the
  smoke expectation through app-level composition rather than weakening Next.js coverage.
- Risk: TanStack route generation changes `routeTree.gen.ts`. Mitigation: allow the generated file
  to change as part of route additions, but do not hand-edit it.
- Risk: server route handler signatures differ from Oberon Handler signatures. Mitigation: keep the
  adapter function tiny and covered by the Handler smoke assertion.
